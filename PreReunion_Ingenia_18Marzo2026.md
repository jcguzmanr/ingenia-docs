## El modelo que nos define — y el problema que resuelve

Antes de hablar de estructura, vale la pena recordar por qué hacemos las cosas como las hacemos.

Ingenia no es una agencia. **Es una consultora.** Y esa diferencia tiene consecuencias reales.

Durante años, las agencias de publicidad produjeron sin coherencia: marcas sin negocio, comunicación sin estrategia, ejecución sin dirección. El cliente tampoco les daba acceso a su data real de negocio.

**El principal problema de las agencias: la desconexión.**
Entre negocio, marca y comunicación. Entre lo que el cliente necesita y lo que se produce.

**Ingenia existe para resolver eso. La coherencia es nuestra clave.**

```
NEGOCIO  →  MARCA  →  COMUNICACIÓN  →  CREATIVIDAD
                                              ↓
                                        OPTIMIZACIÓN
                                              ↓
                                        APRENDIZAJE  ↺ (vuelve al inicio)
```

Esta secuencia no es un capricho — es un sistema circular. Los aprendizajes de cada ciclo retroalimentan el siguiente. Por eso el trabajo que hacemos tiene valor acumulativo.

### ¿Qué significa esto en la práctica?

| Si empezáramos por... | El resultado sería... |
|-----------------------|----------------------|
| Creatividad (sin diagnóstico) | Ejecución sin dirección |
| Comunicación (sin marca) | Mensajes sin identidad |
| Producción (sin estrategia) | Más trabajo, menos impacto |

**El cliente no nos paga por piezas. Nos paga por criterio.**
Criterio para decidir qué hacer, cuándo y por qué. Y luego coherencia para ejecutarlo.

### La analogía que más lo explica

Imagina un partido de fútbol bajo el modelo publicitario viejo: los delanteros se quedan arriba esperando la bola. Todos corretean. No hay bloque, no hay sistema.

El modelo consultivo funciona como el fútbol moderno: **cuando atacas, atacas en bloque. Cuando defiendes, defiendes en bloque.** Duplas, pases limpios, fricción constante para elevar la calidad. Si le pasas mal la posta al creativo, la carrera se pierde desde antes.

## Cómo estamos organizados — el mapa rápido

La organización funciona como un **sistema circular de valor**, no como una jerarquía tradicional:

```
┌─────────────────────────────────────────────────────────┐
│  CREACIÓN DE VALOR                                      │
│  CEO · Dirección de Estrategia                          │
│  → Aquí nace la dirección                               │
├─────────────────────────────────────────────────────────┤
│  GOBERNANZA DEL MODELO  ← nuevo rol en 2026             │
│  Dirección de Operaciones Consultivas & Pipeline        │
│  → Protege que el sistema funcione                      │
├─────────────────────────────────────────────────────────┤
│  OPTIMIZACIÓN                                           │
│  Dirección Creativa · Proyectos Digitales               │
│  → Estrategia → sistemas → activos → ejecución          │
├─────────────────────────────────────────────────────────┤
│  SOSTENIBILIDAD                                         │
│  Administración                                         │
│  → Margen, rentabilidad, cotizadores                    │
├─────────────────────────────────────────────────────────┤
│  APRENDIZAJE  ↺  (cierra el círculo)                    │
│  Control del Valor · Planner Jr                         │
│  → KPIs, insights → retroalimenta el Roadmap            │
└─────────────────────────────────────────────────────────┘
```

Mañana veremos esto en más detalle. Lo importante: **cada capa tiene un mandato diferente** y cuando una invade a otra, el modelo se degrada. La última capa — Aprendizaje — es lo que convierte el modelo en un sistema que mejora con el tiempo.

## Por qué este momento

Cuando una consultora crece, hay dos caminos:

**Camino A:** crecer sin orden → más carga, más fricción, más sobreentregas, pérdida de la promesa al cliente.

**Camino B:** crecer con sistema → el modelo se protege, el equipo tiene menos ruido, los clientes perciben más valor.

Mi ingreso es una apuesta por el **Camino B**.

El fondo es simple: tengo experiencia construyendo sistemas operativos en contextos de crecimiento consultivo, y Ingenia llega a un punto donde ese perfil complementa exactamente lo que ya está funcionando bien.
